Thank you for your interest in contributing to Kapitanov_MRRC.

How to contribute

- Issues: open an issue with a clear title and reproducible steps or data references.
- Pull requests: fork the repository, create a feature branch, and open a PR against `main` with a descriptive title.

Style & tests

- Follow existing code style (PEP8 for Python).
- Include tests for new features where possible and update README if usage changes.

Large files and data

- Do not commit large binary files or raw data to the repo. Use the `results/` folder locally (it is in `.gitignore`), and consider releasing large outputs as GitHub Releases or via external storage.
